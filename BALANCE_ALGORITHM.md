# cyrenenameroller-balance/v3 算法文档

## 基本信息

- **算法名称**：cyrenenameroller-balance/v3
- **引擎版本**：3.1.1
- **算法类型**：Cyrene Adaptive Fairness（CAF）
- **设计目标**：长期公平、短期保留随机性、单次概率受控
- **稳定模块名**：`src/utils/cyrene-balance.js`

模块名不再包含算法版本。以后升级 v4、v5 时只修改模块内部实现和版本常量，调用方无需再次更换文件路径。

## v2 的实际问题

v2 使用：

```text
deficit = count - minCount
weight = max(1, maxRatio - deficit * sensitivity)
```

它存在以下问题：

1. 线性权重在下限 `1` 处截断，大部分人容易堆积到同一最低权重，概率变化不连续。
2. `maxRatio = 5` 时，13 人中单个最高权重者的概率可达到 `5 / (5 + 12) = 29.41%`。
3. 多人最终抽取曾复用同一份历史计数，整批结果没有逐人反馈，不符合马尔科夫状态转移。
4. 禁止重复时曾绕过平衡算法，直接使用均匀随机。
5. “理论最大极差 = (maxRatio - 1) / sensitivity”并不成立。只要所有人始终保留非零概率，随机过程就不能承诺有限的硬极差上限。

## v3 状态模型

普通名单成员构成公平统计总体，白名单成员不计入公平计数。对普通成员 `i`：

```text
totalDraws = sum(count_i)
expected = totalDraws / N
error_i = count_i - expected
```

- `error_i < 0`：低于当前期望次数，提高权重。
- `error_i > 0`：高于当前期望次数，降低权重。
- `error_i = 0`：处于当前公平目标。

当前实现只依赖计数状态、名单、设置和本次排除集合。每次抽取后的新计数是下一次抽取的完整状态，因此满足一阶马尔科夫特性。

## 固定软极差目标

v3.1 将绝对极差 `2` 固定为内部软目标，不在设置页暴露灵敏度、权重倍率或极差阈值参数。这样用户无法通过调参把公平过程变成近似均匀随机或近似必选。

每次计算某个候选者的权重时，算法先模拟“如果下一抽选中此人”后的状态：

```text
projectedGap = gap(count with candidate_i incremented)
```

- 当前极差不超过 2，且该候选者会使极差超过 2：施加温和惩罚，但保留真实的正概率机会。
- 当前极差已经超过 2：对高于最低次数层的候选者施加恢复惩罚，差距越大惩罚越强。
- 最低次数层始终保留正常的正权重。

算法还使用固定的单次概率护栏，避免极端历史把某个人推到近似 100% 的概率。护栏不属于用户设置，也不会写入可编辑配置。

`2` 是适合课堂点名场景的长期产品目标，不是随机过程的数学标准或硬上限。因为所有候选者都必须保留非零概率，任何单次抽取仍可能短暂使极差达到 3 或 4；负反馈会在后续抽取中把状态拉回目标附近。v3.1.1 放松了从极差 2 越界到 3 时的惩罚，避免最高次数层出现近似禁选的极低概率。

## 自适应负反馈

先计算当前极差：

```text
gap = max(count_i) - min(count_i)
gapPressure = clamp(gap / 2, 0, 2)
adaptiveGain = internalSensitivity * (0.35 + 0.65 * gapPressure)
rawLogWeight_i = -adaptiveGain * error_i
```

含义：

- 极差较小时反馈温和，保留随机感。
- 极差增大时自动提高反馈强度，加快回归。
- `internalSensitivity` 是算法内部常量，用户不能修改。

## 内部概率护栏

原始对数权重先经过内部范围裁剪，再乘以软极差惩罚。所有权重始终严格大于零。归一化后，算法对当前候选池执行固定的单次概率护栏：

```text
P(i) <= max(30%, 1 / candidateCount)
```

- 候选者达到 4 人或更多时，任何人当前一抽的概率不超过 30%。
- 候选者只有 1 至 3 人时，使用能让概率总和保持 100% 的数学下限。
- 多人模式不会把 30% 解释为“整批固定抽中 30% 次”；每抽一人后都会更新状态并重新计算下一抽概率。

该护栏控制每一步的最大概率，避免极端历史导致近似必选；它不对随机过程作不成立的硬极差承诺，也不允许用户修改。

## 冷启动

样本不足时，统计误差的可信度较低。v3 使用两轮名单规模作为冷启动区间：

```text
warmup = clamp(totalDraws / (N * 2), 0, 1)
```

- 初始状态接近均匀随机。
- 随着记录增加，反馈平滑增强。
- 不会在第一、第二次抽取后立即产生激进概率偏移。

## 多人模式

### 允许重复

假设一次抽取 `K` 人：

```text
S0 -> 抽取第 1 人 -> 更新临时计数 S1
S1 -> 抽取第 2 人 -> 更新临时计数 S2
...
S(K-1) -> 抽取第 K 人 -> SK
```

每个位置都会重新计算期望、误差、反馈强度和权重。人数没有算法上限，同一人可以在一批中再次出现，但第二次出现前一定已经处理了第一次结果的反馈。

### 禁止重复

流程同样逐人执行，同时将已抽中的名字加入本批排除集合：

```text
抽取 -> 更新临时计数 -> 从本批候选池排除 -> 重新计算 -> 下一人
```

人数上限由界面限制为当前可用名单人数。禁止重复只影响本批候选资格，不会绕过公平权重。

## 随机源

加权选择优先使用 Web Crypto 的 `crypto.getRandomValues()`。只有运行环境不提供 Web Crypto 时才回退到 `Math.random()`。

## 白名单规则

- 白名单成员固定使用中性权重 `1`。
- 白名单抽中后不写入公平计数。
- 白名单仍参与总概率归一化。
- 单次概率护栏对包含白名单在内的整个当前候选池生效。

## 参数

| 参数 | 范围 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | 布尔值 | `true` | 关闭后所有候选者权重均为 1 |
| `algorithm` | 固定字符串 | `cyrenenameroller-balance/v3` | 设置版本标识 |

设置继续保存在 `balance` 键下：

```json
{
  "enabled": true,
  "algorithm": "cyrenenameroller-balance/v3"
}
```

检测到旧算法标识或旧参数字段时，只保留启用/禁用状态，并迁移到 v3.1 的固定内部参数。

公平算法依赖持续、完整的抽取计数。启用公平时，应用会自动启用统计并锁定统计开关；关闭公平后，用户才能单独关闭统计。这样可避免因停止记录状态而让后续概率基于过期计数。

## 稳定 API

```javascript
import {
  computeCyreneBalanceProbability,
  pickCyreneBalanced,
  pickCyreneBatch
} from './cyrene-balance'
```

- `computeCyreneBalanceProbability`：计算当前完整名单的下一抽概率。
- `pickCyreneBalanced`：执行一次有放回或带排除集合的抽取。
- `pickCyreneBatch`：按逐抽反馈语义执行多人抽取。

## 公平性边界

v3 在以下两点之间作明确取舍：

1. 每个人始终保留非零概率，结果仍然是随机抽取。
2. 每一步的单人概率有固定护栏，避免单人因纠偏获得失控概率。

因此，v3.1 让极差形成稳定的均值回归过程，并把长期极差压在很小范围；在固定种子模拟中，13 人 10 万次抽取的最终极差稳定为 1-2，但不宣称每一个瞬时状态都绝不会超过 2。若要求绝对硬极差，只能在阈值触发时将部分成员概率设为零，这与“所有人始终有机会”冲突。

## 版本历史

### v3.1.1

- 保留绝对极差 2 作为固定软目标。
- 将边界越界惩罚从近似硬墙调整为温和负反馈。
- 在 13 人、极差 2 的典型状态下，将最高次数层概率从约 0.11% 提升到约 1.04%。
- 10 万抽固定种子模拟中最终极差仍保持在 1-2。

### v3.1.0

- 将绝对极差 2 固定为不可修改的软目标。
- 增加越界预测惩罚、失衡恢复惩罚和内部概率护栏。
- 移除用户可修改的灵敏度、最大权重比和曲线参数。

### v3.0.0

- 使用期望次数误差作为公平目标。
- 加入随极差变化的自适应负反馈。
- 使用指数权重并对全局权重比作硬裁剪。
- 加入平滑冷启动。
- 使用 Web Crypto 随机源。
- 修复允许重复多人模式未逐抽更新状态的问题。
- 修复禁止重复多人模式绕过平衡算法的问题。
- 将模块固定命名为 `cyrene-balance.js`。

### v2.1.0

- 使用线性截断权重。
- 提供 `sensitivity` 和 `maxRatio` 参数。

### v2.0.0

- 使用基于平方差的倒数权重。

### v1.0.0

- 使用二次插值平衡曲线。
